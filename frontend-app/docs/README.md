# Project Documentation

This directory contains all technical and project management documentation for the Medialternatives headless CMS project. It serves as a central knowledge base for developers, designers, and project stakeholders.

## Directory Structure

- **[architecture/](./architecture/)**: Contains documents related to the high-level architecture of the application, including system design, component diagrams, and data flow.
- **[auth/](./auth/)**: Documentation regarding authentication and authorization strategies.
- **[case/](./case/)**: Case studies and specific use-case documentation.
- **[deployment/](./deployment/)**: Guides and configuration for deploying the application to Vercel and Cloudflare, including CI/CD pipelines and environment setup.
- **[features/](./features/)**: Detailed specifications and implementation notes for various application features.
- **[google/](./google/)**: Information on integration with Google services like Google Analytics 4 and AdSense.
- **[guides/](./guides/)**: Practical how-to guides for developers, covering topics like local development setup, common workflows, and troubleshooting.
- **[jsdoc/](./jsdoc/)**: Information related to the JSDoc setup and generated API documentation.
- **[project/](./project/)**: Project management documents, including the migration plan, sprint tasks, and development standards.
- **[seo/](./seo/)**: Documentation on the SEO strategy, including meta tags, structured data, and performance optimization for search engines.
- **[wordpress/](./wordpress/)**: Details on the integration with the WordPress.com API, including endpoint usage, data models, and content migration notes.

## Contributing

We encourage all team members to contribute to the documentation. When adding or updating a feature, please ensure the corresponding documentation is also updated. Create new markdown files within the relevant subdirectories.
